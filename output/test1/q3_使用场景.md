# 问题3：什么情况下会使用任务管理工具？

## 回答

### 应该使用任务管理工具的场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **复杂多步骤任务** | 任务需要 3 个或更多独立步骤 | "重构认证系统并添加测试" |
| **用户提供多个任务** | 用户一次性给出多项工作 | "1. 修复登录 bug 2. 添加日志 3. 更新文档" |
| **规划模式** | 进入 Plan Mode 后跟踪实施进度 | 设计一个新功能的实施路径 |
| **用户明确要求** | 用户直接要求使用 todo list | "帮我列一个任务清单" |
| **需要跟踪进度** | 需要向用户展示工作进展 | 涉及多文件修改的大型重构 |
| **存在任务依赖** | 某些工作必须在其他工作完成后才能开始 | "先搭建数据库，再写 API，最后写前端" |

### 不应该使用任务管理工具的场景

| 场景 | 说明 | 示例 |
|------|------|------|
| **单一简单任务** | 只有一个直接的操作 | "修复这个 typo" |
| **少于 3 步的简单任务** | 不需要组织管理 | "在函数里加一行日志" |
| **纯对话/信息查询** | 不涉及执行动作 | "解释一下这段代码" |

### 使用时机的最佳实践

1. **收到新指令后**：立即将用户需求拆解为任务
2. **开始工作前**：将任务标记为 `in_progress`
3. **完成工作后**：标记为 `completed`，并检查是否有新的可执行任务
4. **发现新工作时**：在实施过程中发现的后续工作，创建新任务追踪
5. **遇到阻塞时**：创建阻塞任务，建立依赖关系
