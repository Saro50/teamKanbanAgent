# 问题4：任务工具的信息会被记录到哪里？能否被用于回顾上下文？

## 回答

### 信息记录位置

任务管理工具的数据存储在 **当前会话的内存中**（session memory），具体来说：

| 存储层 | 说明 |
|--------|------|
| **会话上下文** | 任务列表作为对话上下文的一部分存在，在当前会话中持续可用 |
| **工具调用历史** | 每次 TaskCreate / TaskUpdate / TaskGet / TaskList 的调用和返回结果都保留在对话历史中 |

### 能否用于回顾上下文？

**在当前会话内：可以。**
- 通过 `TaskList` 查看所有任务的状态概览
- 通过 `TaskGet` 获取特定任务的完整描述和依赖关系
- 任务的 `description` 字段可以记录丰富的上下文信息
- 任务的 `metadata` 字段可以附加任意自定义键值对

**跨会话：不能直接持久化。**
- 任务数据不会自动保存到文件系统或数据库
- 会话结束后，任务列表会随会话上下文一起消失
- 如果需要持久化，需要**手动将任务信息导出到文件**（如本次操作将结果输出到 output 目录）

### 局限性

1. **无远端同步**：任务数据仅存在于本地会话，无法在团队间共享
2. **无持久化存储**：重启会话后任务数据丢失
3. **无历史版本**：任务状态变更没有审计日志
4. **无通知机制**：任务状态变化不会主动通知相关方

### 如何弥补

若需要持久化和共享任务信息，可以考虑：
- 将任务导出为 JSON/Markdown 文件，提交到 Git 仓库
- 结合外部数据库或项目管理工具（如 Jira、Linear、GitHub Issues）
- 通过 MCP Server 扩展对接外部存储
