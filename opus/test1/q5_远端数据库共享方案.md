# 问题5：设计任务处理上下文共享到远端数据库的方案

## 需求分析

**核心目标**：将 Agent 的任务处理上下文（任务信息、执行过程、规划结果）同步到远端数据库，实现团队内任务处理结果和规划信息的共享。

## 方案设计

### 核心模块设计

#### 模块1：Context Sync Middleware（上下文同步中间件）

**职责**：拦截 Agent 的任务操作，收集上下文信息，推送到远端。

**实现方式**：通过 Claude Code 的 Hooks 机制，在每次任务工具调用后触发同步。

```jsonc
// .claude/settings.json - Hooks 配置
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "TaskCreate|TaskUpdate|TaskGet|TaskList",
        "command": "node ./scripts/sync-task-context.js"
      }
    ]
  }
}
```

**同步数据结构**：

```typescript
interface TaskSyncPayload {
  // 基础信息
  sessionId: string;          // 会话 ID
  agentId: string;            // Agent 标识
  timestamp: string;          // ISO 时间戳
  operator: string;           // 操作人员

  // 任务信息
  task: {
    id: string;
    subject: string;
    description: string;
    status: 'pending' | 'in_progress' | 'completed' | 'deleted';
    owner?: string;
    blocks?: string[];
    blockedBy?: string[];
    metadata?: Record<string, any>;
  };

  // 执行上下文
  context: {
    action: 'create' | 'update' | 'query';
    previousStatus?: string;   // 状态变更前
    toolCallHistory?: string[];// 相关工具调用摘要
    relatedFiles?: string[];   // 涉及的文件列表
    summary?: string;          // AI 生成的执行摘要
  };
}
```

#### 模块2：Sync Server（同步服务）

**技术栈**：Node.js + Express/Fastify + PostgreSQL + Redis

**核心 API**：

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/tasks` | 创建/更新任务 |
| GET | `/api/tasks` | 查询任务列表（支持筛选） |
| GET | `/api/tasks/:id` | 获取任务详情及完整上下文 |
| GET | `/api/tasks/:id/timeline` | 获取任务处理时间线 |
| GET | `/api/sessions/:id/context` | 获取会话完整上下文 |
| WebSocket | `/ws/tasks` | 实时任务状态推送 |

#### 模块3：数据存储设计

**PostgreSQL 表结构**：

```sql
-- 任务主表
CREATE TABLE tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id VARCHAR(255) NOT NULL,
    agent_task_id VARCHAR(50) NOT NULL,
    subject TEXT NOT NULL,
    description TEXT,
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    owner VARCHAR(255),
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 任务依赖关系表
CREATE TABLE task_dependencies (
    task_id UUID REFERENCES tasks(id),
    depends_on UUID REFERENCES tasks(id),
    PRIMARY KEY (task_id, depends_on)
);

-- 任务上下文时间线
CREATE TABLE task_timeline (
    id SERIAL PRIMARY KEY,
    task_id UUID REFERENCES tasks(id),
    action VARCHAR(50) NOT NULL,
    context JSONB NOT NULL,
    operator VARCHAR(255),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 会话上下文
CREATE TABLE session_context (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id VARCHAR(255) NOT NULL,
    agent_id VARCHAR(255),
    summary TEXT,
    related_files TEXT[],
    tool_call_count INT DEFAULT 0,
    started_at TIMESTAMPTZ DEFAULT NOW(),
    ended_at TIMESTAMPTZ
);
```

### 团队共享功能

#### 1. 实时看板视图
- 通过 WebSocket 推送任务状态变化
- 团队成员可实时看到 Agent 的任务处理进展

#### 2. 上下文回溯
- 任何团队成员可以查看某个任务的完整处理时间线
- 包括：创建原因、状态变更历史、涉及的文件、执行摘要

#### 3. 跨会话知识沉淀
- 历史任务的处理方案和上下文被永久保存
- 新会话可以查询历史任务，复用已有方案
