# 问题6：规划带人工审核的任务看板流程

## 需求理解

需要一个任务看板系统，其中：
- **人可以审核**由 Agent 生成的任务规划
- **人可以编辑**任务内容和优先级
- **人决定**哪些任务可以被实际执行
- 形成一个完整的 **Agent 规划 → 人工审核 → 执行** 闭环流程

## 流程设计

```
│  规划任务     │──▶│  看板编辑     │──▶│  执行任务     │──▶│  确认完成     │
│              │    │              │    │              │    │              │
│ • 分析需求   │    │ • 查看任务   │    │ • 按优先级   │    │ • 审查产出   │
│ • 拆解任务   │    │ • 编辑修改   │    │   逐个执行   │    │ • 确认通过   │
```


### 状态流转规则

```
                    ┌─── 拒绝(附理由) ──→ 已删除/已归档
                    │
Draft ──→ 人工审核 ─┼─── 需修改 ──→ Editing ──→ 重新提审 ──→ 人工审核
                    │
                    └─── 批准 ──→ Approved ──→ Running ──→ Review ──→ Done
                                                             │
                                                             └── 打回 ──→ Running
```

### 角色与权限

| 角色 | 权限 |
|------|------|
| **Agent（AI）** | 创建任务草稿、执行已批准任务、报告进度、提交执行结果 |
| **审核人（Reviewer）** | 查看所有任务、编辑任务内容、批准/拒绝任务、调整优先级和依赖关系 |
| **管理员（Admin）** | 全部权限 + 配置看板、管理成员、查看分析报表 |

### 看板交互设计

#### 任务卡片信息

```

│─────────────────────────────────│

│ 创建: 2026-03-20 10:30         │
│ 预估: 2h                        │
│─────────────────────────────────│
│ 依赖: #12 数据库 Schema 设计    │
│ 阻塞: #14 前端登录页面          │
│─────────────────────────────────│
│ [查看详情] [编辑] [批准] [拒绝] │
└─────────────────────────────────┘
```

#### 审核操作

1. **批准（Approve）**：任务进入 Approved 列，等待 Agent 执行
2. **拒绝（Reject）**：附加拒绝理由，任务归档
3. **请求修改（Request Changes）**：附加修改意见，任务进入 Editing 状态
4. **编辑（Edit）**：直接修改任务的标题、描述、优先级、依赖关系
5. **合并/拆分**：合并相似任务 或 将大任务拆分为子任务

### 通知机制

| 事件 | 通知对象 | 通知方式 |
|------|----------|----------|
| Agent 创建新任务 | 审核人 | 看板推送 + 消息通知 |
| 任务被批准 | Agent | 触发执行 |
| 任务被打回 | Agent | 触发修改 |
| Agent 执行完成 | 审核人 | 看板推送 + 消息通知 |
| 验收通过/打回 | Agent / 团队 | 状态更新通知 |

### 技术实现要点

| 组件 | 方案 |
|------|------|
| 看板前端 | React + DnD Kit（拖拽）+ WebSocket（实时更新） |
| 后端 API | 复用问题5的 Sync Server，扩展审核流 API |
| 状态机 | XState 或自定义状态机管理任务流转 |
| 通知 | WebSocket 实时推送 + 邮件/Webhook 异步通知 |
| 权限 | JWT + RBAC 中间件 |
