# 问题1：用三句话概括 Git rebase 和 merge 的区别

> 约束：不用代码示例，不用比喻。

## 回答

1. **merge** 会创建一个新的合并提交来整合两条分支的历史，保留完整的分叉与汇合记录；**rebase** 则将当前分支的提交逐个"重放"到目标分支顶端，生成新的提交哈希，形成一条线性历史。

2. merge 是非破坏性操作，不会改写已有提交，适合多人协作的公共分支；rebase 会改写提交历史，因此只应在尚未推送到远端的本地分支上使用，否则会导致协作者的历史冲突。

3. 选择上，团队协作和需要追溯合并时间点时用 merge，追求干净线性历史和便于代码审查时用 rebase。
