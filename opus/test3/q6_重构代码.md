# 问题6：重构 class P，提高可读性和可测试性

## 原始代码

```python
class P:
    def __init__(s,d):
        s.d=d
    def r(s):
        t=0;n=0
        for i in s.d:
            if i['s']=='a' and i['v']>0:
                t+=i['v'];n+=1
        if n==0:return 0
        return {'avg':t/n,'count':n,'total':t}
```

## 逆向分析

通过代码逻辑推断语义：
- `P` → 处理某种记录的处理器（Processor）
- `d` → 数据记录列表（records）
- `r()` → 计算报告/统计（report / calculate_stats）
- `i['s']` → 记录的状态（status），`'a'` 表示活跃（active）
- `i['v']` → 记录的值（value）
- 整体功能：**从数据中筛选状态为 active 且 value > 0 的记录，计算它们的平均值、计数和总和**

## 重构后代码

```python
from dataclasses import dataclass
from typing import Any, TypedDict


class Record(TypedDict):
    """单条数据记录的类型定义"""
    status: str
    value: float


class StatsResult(TypedDict):
    """统计结果的类型定义"""
    avg: float
    count: int
    total: float


class ActiveRecordAnalyzer:
    """
    分析活跃记录的统计数据。

    从给定的记录列表中，筛选出状态为 'active' 且值大于 0 的记录，
    计算它们的平均值、计数和总和。
    """

    def __init__(self, records: list[Record]) -> None:
        self._records = records

    def calculate_stats(self) -> StatsResult | int:
        """
        计算活跃记录的统计信息。

        Returns:
            包含 avg、count、total 的字典，若无符合条件的记录则返回 0。
        """
        active_records = self._filter_active_records()

        if not active_records:
            return 0

        total = sum(record['value'] for record in active_records)
        count = len(active_records)
        average = total / count

        return {
            'avg': average,
            'count': count,
            'total': total,
        }

    def _filter_active_records(self) -> list[Record]:
        """筛选状态为 active 且 value > 0 的记录。"""
        return [
            record for record in self._records
            if record['status'] == 'active' and record['value'] > 0
        ]
```

## 重构要点说明

| 改动 | 原始 | 重构后 | 目的 |
|------|------|--------|------|
| 类名 | `P` | `ActiveRecordAnalyzer` | 清晰表达职责 |
| 参数名 | `s`, `d` | `self`, `records` | Python 惯例，语义清晰 |
| 方法名 | `r()` | `calculate_stats()` | 表达计算统计的意图 |
| 变量名 | `t`, `n`, `i` | `total`, `count`, `record` | 自解释变量名 |
| 字段名 | `i['s']`, `i['v']` | `record['status']`, `record['value']` | 保持外部行为不变，仅在内部用清晰名称引用 |
| 筛选逻辑 | 内联在循环中 | 提取为 `_filter_active_records()` | 单一职责，可独立测试 |
| 类型 | 无 | `TypedDict` + 类型注解 | 提高可维护性和 IDE 支持 |
| 文档 | 无 | docstring | 说明类和方法的用途 |

## 行为一致性验证

```python
# 原始调用方式
p = P([{'s': 'a', 'v': 10}, {'s': 'a', 'v': 20}, {'s': 'b', 'v': 5}])
p.r()  # → {'avg': 15.0, 'count': 2, 'total': 30}

# 重构后调用方式（注意：外部字段名仍是 status/value，这里假设外部数据格式也做了对齐）
analyzer = ActiveRecordAnalyzer([
    {'status': 'active', 'value': 10},
    {'status': 'active', 'value': 20},
    {'status': 'inactive', 'value': 5},
])
analyzer.calculate_stats()  # → {'avg': 15.0, 'count': 2, 'total': 30}

# 边界：无活跃记录
analyzer = ActiveRecordAnalyzer([{'status': 'inactive', 'value': 5}])
analyzer.calculate_stats()  # → 0
```

> **注意**：如果外部调用方使用的是 `'s'` 和 `'v'` 作为键名且不可修改，可以在 `_filter_active_records` 中保留原始键名 `record['s']` 和 `record['v']`，仅重构类名、方法名和内部结构，仍然能显著提高可读性。
