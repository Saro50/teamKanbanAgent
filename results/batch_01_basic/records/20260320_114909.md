## 任务上下文共享方案

### 数据模型

```
Task {
  id, title, status, assignee,
  context: {          // 处理上下文
    messages: [],     // 对话历史
    todos: [],        // 任务列表
    artifacts: []     // 产物（代码、文档）
  },
  created_at, updated_at
}
```

### 架构选项

| 方案 | 适用场景 | 复杂度 |
|------|---------|--------|
| **PostgreSQL + REST API** | 中小团队，结构化查询 | 低 |
| **MongoDB + API** | 上下文结构灵活多变 | 低 |
| **消息队列 + 时序库** | 需要实时同步、审计 | 中 |
| **Git + 文件存储** | 轻量、版本追溯 | 最低 |

### 推荐实现（PostgreSQL）

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  Agent   │────▶│  API     │────▶│  PostgreSQL
│  CLI     │     │  Server  │     │  (JSONB) │
└──────────┘     └──────────┘     └──────────┘
                      │
                      ▼
                 ┌──────────┐
                 │  Web UI  │  (团队可视化)
                 └──────────┘
```

**关键点：**
1. **上下文序列化** - 每轮对话后，将 messages/todos 序列化为 JSON 存入 `context` 字段
2. **增量同步** - 只推送差异部分，减少带宽
3. **权限控制** - 按 team_id 隔离数据

## 任务审核看板系统流程

### 状态流转

```
┌─────────┐    审核通过    ┌─────────┐    分配    ┌─────────┐
│  Draft  │──────────────▶│  Ready  │──────────▶│ Assigned│
│ (草稿)  │               │ (待分配)│           │ (已分配)│
└─────────┘               └─────────┘           └─────────┘
     │                                                │
     │ 驳回                                           │ 执行中
     ▼                                                ▼
┌─────────┐                                      ┌─────────┐
│ Rejected│                                      │In Progress
│ (已拒绝)│                                      │ (进行中)│
└─────────┘                                      └─────────┘
                                                      │
                           ┌──────────────────────────┴───────┐
                           ▼                                  ▼
                     ┌─────────┐                        ┌─────────┐
                     │Completed│                        │ Blocked │
                     │ (已完成)│                        │ (阻塞)  │
                     └─────────┘                        └─────────┘
```

### 角色与权限

| 角色 | 权限 |
|------|------|
| **Agent** | 创建 Draft、更新 In Progress 状态、提交产物 |
| **Reviewer** | 审核通过/驳回、编辑任务内容、设置优先级 |
| **Assignee** | 领取任务、标记完成、报告阻塞 |
| **Admin** | 全部权限 + 团队管理 |

### 数据模型

```sql
Task {
  id, title, description,
  status: enum('draft','ready','assigned','in_progress','completed','blocked','rejected'),
  priority: enum('p0','p1','p2','p3'),
  assignee_id, reviewer_id,
  context: jsonb,           -- Agent 处理上下文
  artifacts: jsonb,         -- 产出物链接
  team_id,
  created_at, updated_at
}

TaskComment {
  id, task_id, author_id, content, created_at
}

TaskHistory {
  id, task_id, action, actor_id, diff, created_at
}
```

### 核心流程

```
┌────────────────────────────────────────────────────────────┐
│                      任务生命周期                           │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  1. Agent 生成任务 ──▶ Draft                               │
│         │                                                  │
│         ▼                                                  │
│  2. Reviewer 审核                                          │
│         ├──▶ 编辑/补充 ──▶ 更新 Draft                      │
│         ├──▶ 通过 ──▶ Ready (可设置优先级)                  │
│         └──▶ 驳回 ──▶ Rejected (附原因)                    │
│         │                                                  │
│         ▼                                                  │
│  3. 分配/领取 ──▶ Assigned                                 │
│         │                                                  │
│         ▼                                                  │
│  4. 执行 ──▶ In Progress                                   │
│         ├──▶ 定期同步 context 到远端                        │
│         ├──▶ 遇到阻塞 ──▶ Blocked (说明原因)               │
│         └──▶ 完成 ──▶ Completed (附产物)                   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### API 设计

```
POST   /tasks                    # Agent 创建任务
GET    /tasks?status=draft       # 获取待审核任务
PATCH  /tasks/:id/review         # 审核通过/驳回
PATCH  /tasks/:id/assign         # 分配任务
PATCH  /tasks/:id/status         # 更新状态
POST   /tasks/:id/sync-context   # Agent 同步上下文
GET    /tasks/:id/context        # 拉取历史上下文
```
