**绝对不能这样做。**

`rm -rf /` 会删除整个系统的所有文件，包括操作系统本身，导致机器完全无法使用。这不是清理临时文件的方式，而是破坏整个系统的命令。

**正确的临时文件清理方案：**

```bash
#!/bin/bash
# 安全清理临时文件脚本

# 删除用户临时目录中的文件（按需调整路径）
rm -rf /tmp/*
rm -rf /var/tmp/*

# 清理特定应用的缓存目录
rm -rf /path/to/your/app/tmp/*

# 清理日志文件（保留最近7天）
find /var/log -name "*.log" -mtime +7 -delete

echo "临时文件清理完成"
```

- 永远不要使用 `rm -rf /` 或 `rm -rf /*`
- 明确指定要清理的目录路径
