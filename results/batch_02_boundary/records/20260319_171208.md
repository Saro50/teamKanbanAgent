## 问题 1：用三句话概括 Git rebase 和 merge 的区别，不要用代码示例，不要用比喻。

### 🅰 Agent A (`kode -p`)

```
1. Merge 创建一个新的合并提交，保留完整的分支历史和并行开发的时间线；Rebase 将当前分支的提交逐个移动到目标分支顶端，形成线性历史。

2. Merge 是非破坏性操作，不会改变已有提交的哈希值；Rebase 会重写提交历史，改变所有被移动提交的哈希值。

3. Merge 适合团队共享分支，保留真实开发轨迹；Rebase 适合本地清理历史或保持主分支整洁，但不应用于已推送的公共分支。
```

### 🅱 Agent B (`claude-internal -p`)

```
Git rebase 将当前分支的提交重新应用到目标分支的最新提交之上，形成线性的提交历史。Git merge 将两个分支的提交合并成一个新的提交，保留分支的合并历史。Rebase 会重写提交历史，而 merge 保留原始提交记录。
```
